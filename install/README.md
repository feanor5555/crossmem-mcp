# crossmem install/ — LLM-addressed install guides

This directory contains **LLM-addressed** install instructions for crossmem.
The audience is not a human reader but the language model running inside one
of the twelve supported MCP-capable coding CLIs (Claude Code, Cursor, Cline,
OpenCode, Pi, Kilo Code, Continue.dev, Gemini CLI, Goose, Windsurf,
Amazon Q CLI, Zed).

For the master orchestration skill that an LLM should load first, see
`skills/crossmem-install/SKILL.md` in the repo root (Task 15.2). The skill
references the per-CLI files listed below via progressive disclosure.

## Contents

- `_template.md` — schema and skeleton every per-CLI install guide must
  follow. The fixed `## ` heading order is:
  **Prerequisites / Install / Configure MCP / Verify / Troubleshooting**.
  Optional YAML frontmatter carries the CLI name, minimum crossmem version,
  platform-specific config paths, and a restart hint.
- `<cli>.md` — one file per supported CLI, generated by
  `crossmem docs install --cli <name>` (Task 15.4) so the docs stay in
  lockstep with the connector code. These files are added in Task 15.5.

## Conventions for new install guides

Every `install/<cli>.md` MUST:

1. Open with a YAML frontmatter block whose keys are exactly the ones
   listed in `_template.md`.
2. Contain the five mandatory `## ` headings in the same order.
3. Be **idempotent** — running the same commands twice must leave the
   system in the same state.
4. End the **Verify** section with `crossmem doctor` returning exit 0
   plus an MCP round-trip the LLM can observe.

A snapshot test in `tests/install/` enforces (1) and (2) for the template
today and will be extended to every `install/*.md` file once Task 15.5
adds them.
